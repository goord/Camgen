17 /* * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * *
18  * Utility class templates with the purpose of defining implementation classes *
19  * from the model classes static data. Every model class should at least contain *
20  * a type definition of the numerical type called 'value_type', for which the *
21  * usual arithmetic operations and std::sqrt should be defined, a const static *
22  * int called 'dimension', denoting the spacetime dimension, and a const static *
23  * bool called 'coloured' denoting whether the model contains unbroken *
24  * nonabelian symmetries. If the dimension is bigger than zero, a spacetime type *
25  * must by given, and a static boolean called 'continuous_helicities' denoting *
26  * whether the external states are generally not helicity eigenstates. Moreover, *
27  * if fermions are present, a definition of the Dirac algebra basis should be *
28  * contained in the model as well as a const static integer 'beam_direction', *
29  * and optionally if these are massive, a definition of the spin vector type. On *
30  * the other hand, if the 'coloured' boolean is true, a const static boolean *
31  * called 'continuous_colours', denoting whether the colour states are *
32  * eigenstates or not. Finally, one should then also define a type called *
33  * 'colour treatment', which determines what colour basis is used by Camgen. *
34  * *
35  * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * */
